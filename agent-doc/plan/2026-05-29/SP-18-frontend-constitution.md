## 最终结果（来自结果蓝图）

### 要实现的内容

#### 评估入口页

要素：
- 最新体质报告卡：体质类型 + 分数 + 趋势箭头（↑/↓/→）+ 较上次变化
- 体质变化趋势图（折线图，按时间）
- 历史评估列表（按时间倒序，每项显示日期+体质+分数）
- 开始新评估按钮 → 选择方式弹窗（问卷评估 / AI对话评估）

#### 体质评估问卷页

线框图：
```
┌─────────────────────────────┐
│ < 体质评估          [退出]  │
│ ████████░░░░ 60%            │
│ ┌─────────────────────┐    │
│ │  第 6/10 题           │    │
│ │  您是否经常感到        │    │
│ │  口干舌燥、咽喉干痛？   │    │
│ │                      │    │
│ │  ○ 没有（1分）         │    │
│ │  ○ 偶尔（2分）         │    │
│ │  ○ 经常（3分）         │    │
│ │  ○ 总是（4分）         │    │
│ └─────────────────────┘    │
│     [←上一题] [下一题→]     │
└─────────────────────────────┘
```

交互元素表：
| 元素 | 类型 | 说明 |
|------|------|------|
| 返回按钮 | IconButton | 返回上一页 |
| 退出按钮 | Text Button | 退出评估（确认弹窗） |
| 进度条 | ProgressBar | 草本绿渐变，当前题数/总题数 |
| 题号 | Text | "第 X/10 题" |
| 题目卡片 | Card | 大圆角卡片，展示题目文本 |
| 选项按钮 | Radio | 4个选项（没有1/偶尔2/经常3/总是4分） |
| 上一步按钮 | Button | 回到上一题（第1题隐藏） |
| 下一步按钮 | Button | 进入下一题（最后1题显示"提交"） |

交互规则：
| 操作 | 行为 |
|------|------|
| 选择选项 | 自动选中，不自动跳转，需用户点"下一步" |
| 点击上一步 | 回到上一题，保留已选答案 |
| 点击下一步 | 校验当前题已选 → 进入下一题 |
| 最后一步 | 按钮变为"提交" → 调 POST /constitution/assess |
| 提交成功 | 跳转评估报告页 |
| 退出 | 弹窗提示"退出后进度将丢失" |

交互元素表：
| 元素 | 类型 | 说明 |
|------|------|------|
| 消息列表 | ScrollView | AI和用户消息气泡交替展示 |
| AI消息气泡 | Bubble | 左侧显示，带🤖头像 |
| 用户消息气泡 | Bubble | 右侧显示，带🙋头像 |
| 输入框 | TextInput | 多行输入 |
| 发送按钮 | Button | 发送消息 |
| 转问卷按钮 | Button | 点击跳转问卷模式 |
| 结束按钮 | Button | 结束对话→调 POST /constitution/chat/assess |

交互规则：
| 操作 | 行为 |
|------|------|
| 发送消息 | 调 POST /constitution/chat (SSE) → 逐字展示 AI 回复 |
| SSE 流式 | fetchEventSource 逐段解析，打字机效果展示 |
| 结束问诊 | 调 POST /constitution/chat/assess → 跳转报告页 |
| 转问卷 | 保留对话 context → 跳转问卷模式 |
