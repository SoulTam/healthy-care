# 中医食补应用 — 项目需求文档

## 1. 项目背景与愿景

中医食补通过饮食调理身体，根据个人体质和季节变化选择适宜食物，达到养生保健的目的。例如，夏季多食清凉解暑之物（绿豆汤、西瓜等），冬季宜温热补气（羊肉汤、红枣等）。中医食补的灵活性和个性化是其核心优势。

本应用旨在将中医食补智慧数字化、个性化、普惠化。核心理念：**以给定的食谱和中医名著为唯一知识来源，做多维索引和智能检索组合，不做超越资料的推理**。让不具备中医基础知识的普通用户也能通过饮食改善健康状况。

## 2. 目标用户

| 属性 | 说明 |
|------|------|
| 用户范围 | 面向所有人群，不限年龄、性别、健康状况 |
| 中医基础 | 用户无需具备任何中医基础知识 |
| 健康状态 | 覆盖健康人群（养生保健）、亚健康人群（调理改善）、慢性病人群（辅助食补，非替代医疗） |

## 4. 核心功能模块

### 4.1 体质评估

| 功能点 | 详细说明 |
|--------|----------|
| 评估方式 | 问卷自评 + AI对话式问诊，两者结合得出综合体质判定 |
| 体质分类 | 基于中医九种体质分类（平和质、气虚质、阳虚质、阴虚质、痰湿质、湿热质、血瘀质、气郁质、特禀质） |
| 多次评估 | 支持用户多次进行体质评估，记录体质变化趋势 |
| AI动态调整 | 日常对话中，AI可根据用户反馈的身体状况及时调整体质判定 |
| 评估报告 | 每次评估生成体质解读报告，包含体质特征说明、调养建议及注意事项 |

### 4.2 食补方案推荐（核心：从已索引资料中检索组合）

| 功能点 | 详细说明 |
|--------|----------|
| 推荐维度 | 综合考虑以下维度从索引库中匹配食谱：用户体质、当前季节、饮食禁忌（过敏/宗教/孕期等）、现有症状（上火/失眠/乏力等）、食材可获取性（地域/时令） |
| 推荐机制 | **基于多维索引检索**：将用户条件转为多维筛选条件 → 在索引食谱库中匹配 → 按匹配度排序 → 组合输出 |
| 方案粒度 | 支持两种粒度：①一日三餐方案 ②一周计划，由用户选择 |
| 单餐替换 | 一周计划中支持单餐一键换菜，系统从索引库中按相同条件检索候补食谱，保证替换后仍符合当前体质和季节需求 |
| 营养数据 | 每道食谱附带营养价值数据（卡路里、蛋白质、脂肪、碳水等），同样来源于已索引资料 |
| 食材可获取性 | 根据用户所在地域和当前时令，优先过滤出易获取、应季的食材 |
| 食谱详情 | 每道食谱包含：食材清单、用量、烹饪步骤、中医功效说明（性味归经、适用体质）、营养数据、禁忌提示 |

## 5. AI技术方案：知识库索引 + 多维检索 + 组合输出

### 5.1 整体架构

核心理念：**用户给什么资料，系统就基于什么资料回答。不做超越资料的推理。**

```
资料输入（食谱/名著）
    ↓ 文本切片 + 元数据标注
结构化食谱库 + 向量索引库
    ↓ 多维标签索引（体质/季节/食材/症状/功效…）
多维索引体系
    ↓ 用户查询 → 多维条件 → 语义检索 + 元数据过滤 → 排序 → 组合
检索结果 → 拼装格式化为方案 → 输出给用户
```

| 环节 | 说明 |
|------|------|
| 资料入库 | 用户投喂食谱（结构化JSON/Markdown）和中医名著原文 → 文本切片 → embedding向量化 → 存入向量数据库 |
| 多维索引 | 每条食谱/知识片段打上多维标签（体质、季节、食材、症状、功效、性味归经、禁忌等），元数据字段化存储，支持精确过滤 |
| 检索阶段 | 先做元数据过滤（如"夏季" + "阴虚体质"）缩小候选池，再做语义检索找到最匹配的内容 |
| 组合输出 | 检索命中多个候选食谱后，按规则（季节匹配度、体质匹配度、营养均衡度）排序，拼装组合成三餐/周方案 |
| 模型角色 | Embedding模型负责向量化（语义检索）；LLM仅用于「自然语言理解用户意图 + 格式化输出结果」，不负责生成新的食补知识 |

### 5.2 资料入库与索引规范

#### 5.2.1 食谱入库格式

每条食谱按统一结构化模板入库：

```yaml
食谱名称: "百合莲子银耳羹"
类别: 甜品/汤羹
体质适应: ["阴虚质", "气虚质"]
季节适应: ["秋季", "冬季"]
食材: ["百合", "莲子", "银耳", "冰糖"]
功效: ["滋阴润肺", "清心安神", "养胃生津"]
性味归经: {百合: "甘微寒，归心肺经", 莲子: "甘涩平，归脾肾心经", 银耳: "甘淡平，归肺胃肾经"}
症状关联: ["干咳", "失眠", "口干", "便秘"]
禁忌: ["风寒咳嗽", "脾胃虚寒泄泻"]
烹饪步骤: "1.银耳泡发撕小朵 2.百合莲子洗净 3.加水炖煮1小时 4.加冰糖调味"
营养成分: {热量: 120, 蛋白质: 3, 脂肪: 0.5, 碳水: 28}
古籍出处: "《随息居饮食谱》"
```

#### 5.2.2 名著入库格式

中医名著按章节/段落切片，每段打标：

```yaml
文本内容: "绿豆甘寒，清热解毒，利水消肿，消暑止渴..."
出处: "《随息居饮食谱》卷三·谷食类"
食材关联: ["绿豆"]
功效关联: ["清热解毒", "消暑", "利水"]
体质关联: ["湿热质", "平和质"]
季节关联: ["夏季"]
```

#### 5.2.3 多维索引体系

| 索引维度 | 类型 | 说明 | 示例值 |
|----------|------|------|--------|
| 体质索引 | 枚举（多选） | 九种体质分类 | 阴虚质、气虚质 |
| 季节索引 | 枚举（多选） | 适宜季节 | 春、夏、秋、冬 |
| 食材索引 | 标签（多选） | 所用食材 | 百合、绿豆 |
| 功效索引 | 标签（多选） | 中医功效 | 滋阴润肺、清热解毒 |
| 性味归经 | 结构化键值对 | 食材→性味归经映射 | 百合→甘微寒，归心肺经 |
| 症状索引 | 标签（多选） | 关联的症状/不适 | 干咳、失眠 |
| 禁忌索引 | 标签（多选） | 禁忌人群/情况 | 风寒咳嗽、孕妇慎用 |
| 餐类索引 | 枚举 | 早/午/晚餐/汤羹/甜品 | 甜品 |
| 来源索引 | 文本 | 典籍出处 | 《随息居饮食谱》 |

### 5.3 检索与组合流程

```
用户输入："我最近口干，经常失眠，秋天了想调理一下"
    ↓ 意图理解（LLM，不做推理）
提取：{症状: [口干, 失眠], 季节: 秋季}
    ↓ 结合用户档案
用户体质: 阴虚质
    ↓ 多维索引检索
查询条件：体质=阴虚质 AND 季节=秋季 AND 症状包含[口干, 失眠]
    ↓ 元数据过滤（精确）+ 语义检索（模糊）
候选食谱池（按匹配度排序）
    ↓ 组合引擎（规则）
筛选优先级：体质完全匹配 > 季节匹配 > 症状匹配
三餐搭配规则：主食 + 主菜 + 汤羹，避免食材重复和性味冲突
    ↓ 格式化输出（LLM）
生成方案卡片 → 返回用户
```

**组合引擎规则示例：**

| 规则 | 说明 |
|------|------|
| 餐类分配 | 早餐→粥/糊类，午餐→主食+热菜+汤，晚餐→轻食/汤羹 |
| 食材去重 | 同一方案内避免食材过度重复 |
| 性味平衡 | 同一餐内避免性味过偏（如全是寒性或全是温热） |
| 优先级 | 体质匹配 > 季节匹配 > 症状匹配 > 功效匹配 |
| 兜底策略 | 若某餐无完全匹配食谱，放宽症状条件重新检索；若仍无，从同体质通用食谱中随机推荐 |

### 5.4 技术选型与部署

#### 5.4.1 开发阶段（本机部署）

硬件配置：AMD Ryzen 7 H 260（8核16线程）+ 32GB内存 + AMD Radeon 780M（无NVIDIA独立GPU）

| 组件 | 选型 | 说明 |
|------|------|------|
| Embedding模型 | BGE-small-zh / BGE-base-zh | 中文语义向量化，CPU即可运行，轻量高效 |
| 向量数据库 | Chroma | 轻量级，Python原生，支持元数据过滤+语义检索，开发阶段足够用 |
| LLM（意图理解+格式化） | Ollama + Qwen2.5-7B-Instruct-Q4_K_M | 本机32GB内存可运行，仅用于自然语言理解和结果格式化，不负责食补推理 |
| 后端框架 | Python FastAPI | 简洁高效，生态成熟 |
| 前端框架 | React / Vue（待定） | 配合App和小程序 |

#### 5.4.2 生产环境部署

| 组件 | 选型 | 说明 |
|------|------|------|
| Embedding模型 | BGE-base-zh（或API服务化部署） | 可升级为BGE-large-zh提升精度 |
| 向量数据库 | Milvus / Chroma生产模式 | 数据量增大后从Chroma迁移至Milvus |
| LLM | Qwen2.5-7B-Instruct AWQ-INT4 | 量化后显存仅需6-8GB，单卡RTX 3060即可 |
| 推理框架 | Ollama（开发）/ vLLM（生产） | 生产环境vLLM提升吞吐量 |

| 规格 | 最低配置 | 推荐配置 |
|------|---------|---------|
| GPU | 无（纯CPU推理） | 1× RTX 3060 12GB |
| 内存 | 16GB | 32GB |
| CPU | 8核+ | 8核+ |
| 存储 | 100GB SSD | 200GB NVMe SSD |

### 5.5 索引维护与优化

```
新增食谱/名著 → 入库切片 → 打标 → 更新索引
                        ↓
              定期检查索引质量（去重、纠错、补充标签）
                        ↓
              用户反馈 → 分析高频匹配失败场景 → 调整索引标签 + 组合规则权重
```

| 维护项 | 说明 |
|--------|------|
| 资料增量更新 | 新增食谱/名著章节 → 自动入库切片 → 自动打标（LLM辅助）→ 人工审核确认 → 更新向量库和元数据库 |
| 标签纠错 | 定期抽查索引标签准确性，发现标注错误及时修正 |
| 权重调优 | 根据用户反馈分析高频不满意场景，调整组合引擎的排序权重和规则 |
| 索引覆盖度监测 | 监控各维度（体质×季节×症状）的食谱覆盖情况，发现空白区提示补充资料 |

## 6. UI设计规范

### 6.1 设计风格：治愈温馨

整体风格定调为**治愈、温馨、中式韵味**，契合中医食补的自然养生调性。

### 6.2 色彩体系

| 用途 | 色值方向 | 示例 | 情感联想 |
|------|----------|------|----------|
| 主色 | 暖米色/奶白色 | `#F5F0E8` | 温润、安宁 |
| 辅助色 | 低饱和暖色 | `#D4A574`（驼色）、`#C9B99A`（麦色） | 自然、质朴 |
| 点缀色 | 草本绿/枸杞红/姜黄 | `#7BA06E`、`#C4704E`、`#D4A843` | 中医食材意象，有生机但不刺激 |
| 文字色 | 深棕/暖灰 | `#4A3B32`、`#6B5E54` | 不用纯黑，避免冷硬感 |
| 背景 | 渐变暖白 | `#FAF7F2 → #F0EDE5` | 有呼吸感，非纯平白 |

### 6.3 核心视觉元素

| 元素 | 设计手法 | 作用 |
|------|----------|------|
| 水墨/工笔风食材插画 | 如枸杞、红枣、山药等以中式工笔淡彩呈现 | 传递中医文化底蕴，避免照片的生硬感 |
| 圆角大卡片 | 卡片式布局，圆角16-24px，柔和阴影 | 信息层次清晰，视觉柔和 |
| 手写体/宋体混排 | 标题用温润手写体，正文用思源宋体 | 中式书卷气+可读性 |
| 留白 | 页面留白率 > 30% | 呼吸感，不拥挤，治愈感的核心 |
| 微动画 | 页面切换淡入淡出，卡片浮现缓慢上移，水波纹点击反馈 | 流动感、生命力，避免突兀跳转 |
